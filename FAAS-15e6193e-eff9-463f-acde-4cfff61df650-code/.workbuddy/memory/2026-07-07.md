## 2026-07-07

### 修复：PDF 附件未出现在运营报告表单

**问题**：全链路测试完成，media_id 已获取、状态已更新为"已完成"，但运营报告表单中附件字段为空。

**根因分析**：
1. **Bug #1（核心）**：`YidaFormUpdater.updateReportRecord()` 中 `formData.put(FIELD_ATTACHMENT, attachmentList.toJSONString())` 将 JSONArray 转成了字符串（带转义引号），导致宜搭 API 收到 `"[{\"fileId\":\"...\"}]"` 而非 `[{"fileId":"..."}]`，无法解析为附件。
2. **Bug #2（潜在）**：`FaasEntry` 后台循环中每次时间范围都调用 `updateReportRecord`，多个时间范围时后一次覆盖前一次。

**修复内容**：
- `YidaFormUpdater.java`：`attachmentList.toJSONString()` → `attachmentList`（直接传 JSONArray）
- `YidaFormUpdater.java`：新增重载方法 `updateReportRecord(formInstId, List<String> mediaIds, List<String> fileNames, ...)`，支持一次更新多个附件
- `FaasEntry.java`：后台任务改为收集所有 media_id/fileName，循环结束后一次性调用 `updateReportRecord` 更新全部附件

**验证**：尚未编译/运行验证（Maven环境问题），代码逻辑经审查正确。
