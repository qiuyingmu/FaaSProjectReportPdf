### 修复：PDF 附件未出现在运营报告表单（傍晚）

**问题现象**：全链路测试成功（media_id 已获取、状态已更新），但表单附件字段为空。

**Bug #1（核心）**：
- 位置：`YidaFormUpdater.updateReportRecord()` 第 213 行
- 原因：`formData.put(FIELD_ATTACHMENT, attachmentList.toJSONString())` 把 JSONArray 转成了**带转义引号的字符串**，宜搭 API 收到的是 `"[{\"fileId\":\"...\"}]"` 而不是 `[{"fileId":"..."}]`，导致无法解析为附件
- 修复：`attachmentList.toJSONString()` → `attachmentList`（直接传 JSONArray）

**Bug #2（潜在）**：
- 位置：`FaasEntry.java` 后台任务循环
- 原因：每次时间范围都调 `updateReportRecord`，多时间范围时后一次覆盖前一次
- 修复：收集所有 media_id/fileName 到 List，循环结束后**一次性调用**重载的 `updateReportRecord(fid, mediaIds, fileNames, ...)`

**YidaFormUpdater 变更**：
- 新增重载方法 `updateReportRecord(formInstId, List<String> mediaIds, List<String> fileNames, ...)`
- 旧单附件接口委托给新方法

### 资料库分类级联路径修复（即时）

**问题**：资料库分类只显示第一级（如 `"监理业务"`），而宜搭页面显示完整级联路径（如 `"监理业务 > 施工单位 > 特种作业人员操作证书及报审表"`）。

**原因**：原代码用 `ReportHelper.extractField()` 提取 `cascadeSelectField`，仅取第一个元素值，且用 `、` 连接。宜搭的级联字段 `value` 是数组，如 `["监理业务", "施工单位", "特种作业人员..."]`，需要用 ` > ` 连接。

**修复**：
- **`ReportHelper.java`**：新增 `extractCascadeLabel(Map, fieldId)` 方法，完全对应 `xm.js` 的 `getCascadeLabel()`：
  - 取 `value` 字段（未定义则取整个字段）→ 转为数组 → 过滤空值 → 将每个元素转为字符串（Map 类型优先取 `key` → `label` → `toString`）→ 用 ` > ` 连接各级
- **`ReportProjectHtmlBuilder.java`** / **`ReportProjectPdfBuilder.java`**：资料库分组调用从 `extractField(fd, "cascadeSelectField_ml6no8w5")` 改为 `extractCascadeLabel(fd, "cascadeSelectField_ml6no8w5")`

**编译**：0 错误，全部编译通过 ✅

### 新建 Spring Boot 定时任务项目（傍晚）

**背景**：宜搭 FaaS 连接器在返回 HTTP 响应后会断开容器资源，不支持异步回调处理。当前两阶段架构存在 PDF 尚未生成就被中断的风险。需要一个独立部署的 Spring Boot 应用，使用 `@Scheduled` 定时任务替代 FaaS 异步处理。

**新项目位置**：`report-springboot/`（与旧 FaaS 项目同级）

**项目结构**：
```

    ├── java/com/alibaba/work/faas/
    │   ├── ReportApplication.java     [NEW]  @EnableScheduling
    │   ├── config/ReportConfig.java   [NEW]  调度线程池
    │   ├── schedule/ReportScheduler.java  [NEW]  3个定时任务
    │   ├── report/                    [COPY] 24个业务文件

    └── resources/ (application.yml + yida-secret + fonts)
```

**定时任务**：
