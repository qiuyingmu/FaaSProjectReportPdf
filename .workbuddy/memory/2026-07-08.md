## 日志中文乱码修复（10:16）

### 问题
实时日志页面中，中文显示为乱码（如 `ä»»å¡` 代替 `任务`）。

### 根因
`LogController.tailFile()` 按字节读取日志文件，并把每个字节直接转为 `(char)`，
导致 UTF-8 多字节中文被当成 Latin-1 解码，从而乱码。

### 修复
- 改为反序收集每一行的字节数组
- 整行反转后使用 `new String(bytes, StandardCharsets.UTF_8)` 解码
- 同时兼容 CRLF 换行

## 前端 401 Session 过期修复（10:32）

### 问题
后端重启后，前端 Session 失效，但所有组件用 `fetch()` 直接请求（没走 axios 拦截器），
导致组件只显示"无法连接"，没有自动跳转登录页。

### 改动
1. **新增 `frontend/src/utils/api.js`** — 统一 API 请求工具
   - `apiFetch()`：封装 fetch，自动携带 credentials，401 时跳转 `/login`
   - `apiGet()` / `apiPost()` / `apiPut()` / `apiDelete()`：便捷的 JSON 响应方法
   - 支持 `noRedirect: true` 选项跳过 401 跳转（用于组件内部静默降级）

2. **更新 `main.js`**：
   - 删除未使用的 axios
   - 路由 `beforeEnter` 守卫：进入 Dashboard 前检查 Session 有效性

3. **更新 5 个组件**全部使用 `apiGet` / `apiPost` 等工具函数：
   - `SystemOverview.vue` / `LogViewer.vue` / `OperationLogs.vue`
   - `ReportGenerator.vue` / `ScheduleManager.vue`
   - 所有 API 调用统一用 `noRedirect: true` + 组件内自行 catch 降级

### 效果
- **后端重启 → 刷新页面 → 路由守卫检测到 401 → 跳转登录页**
- **Session 过期 → 下次 API 请求返回 401 → `apiFetch` 自动跳转登录**
- **正常使用时**：各组件的 catch 逻辑不变，不会误跳
