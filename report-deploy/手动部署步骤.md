# 手动部署步骤

## 部署包内容说明

`report-deploy.tar.gz`（89MB）包含两部分：

| 部分 | 文件 | 部署目标 |
|------|------|---------|
| **后端** | `report-springboot-1.0.0.jar` + `Dockerfile.backend` + `docker-compose.yml` | Docker 容器 `report-backend` |
| **前端** | `frontend/index.html` + `frontend/assets/` | Nginx webroot `/www/wwwroot/realtimevideo.jgjl.cn/report/` |
| **配置** | `.env` + `nginx-subpath.conf` | 两者都需要 |

---

## 🚀 后端部署（Docker）

### 第 1 步：解压

```bash
cd /opt/report
rm -rf frontend/ report-springboot-1.0.0.jar
tar xzf report-deploy.tar.gz
```

### 第 2 步：恢复 .env 配置

```bash
cat .env
# ⚠️ 如果显示的是模板默认值（your-xxx-here），填入你的真实值
```

需要恢复的字段（必不可少）：

| 变量名 | 示例值 | 说明 |
|--------|--------|------|
| `REPORT_ADMIN_PASSWORD` | `你的密码` | 管理员登录密码 |
| `DB_PASSWORD` | `Report@2026` | 数据库密码 |
| `dingtalk.app.key` | `dingagu9xrk9ribreqdf` | 钉钉 AppKey |
| `dingtalk.app.secret` | `sf9qY7R32RxA...` | 钉钉 AppSecret |
| `yida.production.app.type` | `APP_ULZ41JL...` | 宜搭应用类型 |
| `yida.production.system.token` | `MO7668710NE1...` | 宜搭 token |
| `yida.production.user.id` | `0204292046...` | 宜搭用户 ID |

> 📌 **OBS 密钥不需要填**，已在 `yida-secret.properties`（JAR 内）配置，从 `.env` 中移除了。

### 第 3 步：停旧容器 + 启动新容器

```bash
docker compose down
docker compose up -d --build
sleep 10
docker logs report-backend --tail 20
```

### 第 4 步：验证后端

```bash
# 查看启动日志，确认调度器正常
docker logs report-backend --tail 30 | grep -i "动态调度器\|周报\|月报\|季报"
# 应输出：周报: cron=0 0 10 * * MON 🟢 等 3 个任务

# 验证 API
curl -s http://localhost:8082/report/api/test

# 验证时区
docker exec report-backend date
# 输出应为：Mon Jul 13 xx:xx:xx CST 2026（北京时间）
```

## 🌐 前端部署（Nginx 静态文件）

### 第 1 步：复制文件到 Nginx webroot

```bash
cp -rf frontend/* /www/wwwroot/realtimevideo.jgjl.cn/report/
```

### 第 2 步：确认文件更新

```bash
ls -la /www/wwwroot/realtimevideo.jgjl.cn/report/assets/
# 确认 index.html 和 assets/ 都是最新时间
```

### 第 3 步：清除 Nginx 缓存

**宝塔面板操作：**

→ 网站 → realtimevideo.jgjl.cn → 缓存 → 清空缓存

> 如果没启用缓存，跳过这步。

---

## ✅ 最终验证

### 浏览器

1. 打开**无痕窗口 / 隐私模式**
2. 访问 `https://realtimevideo.jgjl.cn/report/`
3. 登录后进入定时任务管理，确认能看到 周报/月报/季报
4. 手动生成一份报告（可选）
5. 确认文件大小正常（不再是 NaNKB）

> 如果还是老版本 → `Ctrl+Shift+Delete` 清除浏览器缓存后重试

### 各任务触发时间（北京时间）

| 任务 | Cron | 触发时间 |
|------|------|---------|
| 周报 | `0 0 10 * * MON` | 每周一 **10:00** |
| 月报 | `0 0 10 1 * ?` | 每月 1 号 **10:00** |
| 季报 | `0 0 10 1 1,4,7,10 ?` | 1/4/7/10 月 1 号 **10:00** |

## ⏪ 回滚方案

### 回滚后端

```bash
docker compose down
# 如果有旧 JAR 备份
cp /opt/report/backup/report-springboot-1.0.0.jar.backup ./report-springboot-1.0.0.jar
docker compose up -d --build
```

### 回滚前端

用之前备份的 `index.html` 和 `assets/` 覆盖 webroot，或重新从旧包解压。
